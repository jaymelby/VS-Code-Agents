Defusing "Agent vs. RPA"

When Risk teams see a chat interface trigger a complex system action, they incorrectly assume the AI "hallucinated" the execution steps. Present the "Intent Router vs. Execution Engine" paradigm: The AI is just the waiter taking the order; the legacy Salesforce Flow is still the kitchen doing the exact same cooking.

The "Waiter to Kitchen" Workflow
1. The Patron

User types unstructured request: "Update address for Acme Corp."

2. The Waiter (LLM)

Agentforce NUI extracts intent and maps it to a specific Action.

PROBABILISTIC
3. The Kitchen (Flow)

LLM hands a structured "ticket" to a pre-approved, deterministic Flow.

DETERMINISTIC
4. The Pass (HITL)

Flow pauses. The human reviews the proposed data change and clicks "Approve".

The Argument for Risk: "If this Salesforce Flow was deemed safe when triggered by a button click in 2022, it is mathematically just as safe today. We only changed the button to a natural language trigger."

Enterprise Architecture

Dual-Stack Topology

Attempting to force a single AI platform across all use cases leads to bloated budgets and compromised security. The modern strategy embraces a Dual-Stack Architecture, utilizing Salesforce Agentforce for CRM-native tasks and Google Vertex for broad, cross-silo enterprise orchestration.

Stack 1: Salesforce Agentforce (The "Micro")

The Mandate: Designed for targeted, contextual workflows directly serving Sales and Relationship Management employees, operating strictly within CRM boundaries.

Strengths: Leverages existing Salesforce RBAC instantly. Highly deterministic for structured tasks (updating records, triggering onboarding sequences) using pre-approved Flows.

Stack 2: Google Vertex + MCP (The "Macro")

The Mandate: The omni-directional brain for complex, multi-hop reasoning that spans the entire enterprise beyond the CRM.

Strengths: Powered by frontier models (Gemini 1.5 Pro). Utilizes the Model Context Protocol (MCP) to securely connect to external APIs, BigQuery, and custom micro-agents.

Protecting the IT Budget

FinOps & The Chargeback Model

If IT pays for all AI compute, the budget will collapse within 12 months. When a specific division (e.g., Wealth Management) uses Vertex to automate 10,000 hours of labor, they reap the P&L rewards—therefore, they must absorb the token costs.

The Token Chargeback Strategy
  • Implement an LLM Gateway (e.g., LiteLLM, Portkey):

    Never let internal apps call Vertex APIs directly. Route all requests through an AI API Gateway. The Gateway standardizes logs, enforces rate limits, and, crucially, intercepts every API call to extract the User_ID and Department_Code.

  • Metadata Tagging for Telemetry:

    Mandate that every Agent deployed on the Enterprise Registry passes metadata headers (e.g., Cost_Center="Wealth_Advisory_North") alongside its payload. If a rogue agent lacks these headers, the Gateway rejects the request.

  • Automated P&L Routing:

    Use Datadog LLM Observability or LangFuse to aggregate these tags. At the end of the month, generate an automated FinOps report that charges the exact token consumption cost back to the respective business unit's OPEX budget.

The "FINRA Day" Protocol

Forensic Auditing & Data Lineage

Three years from now, FINRA will audit a trade recommendation made by an agent. If your answer is "the black box told us to," the firm faces catastrophic fines. You must architect for retroactive determinism.

The Immutable Receipt

Every time an Agentforce action executes or a Vertex MCP sequence completes, it must write a "Forensic Receipt" to WORM (Write Once, Read Many) compliant storage. This receipt must contain:

  • The Prompt Context: The exact system prompt used (e.g., v2.4.1_Wealth_Advisor_Prompt).
  • The Model State: The exact model and weight version (e.g., gemini-1.5-pro-002) and temperature settings.
  • The Data Snapshot: A hash of the specific RAG documents or CRM records the agent read at that exact millisecond. If the policy changed the next day, you can prove the agent acted correctly on the information available at that time.
  • The User State: The active directory token and the exact time the user clicked "Approve" (The HITL signature).

"The goal is not to prove the AI is perfect; the goal is to prove the AI's logic is perfectly traceable."

Crisis Mitigation

The Enterprise Kill Switch

Agents will fail. They will hallucinate a bad process or face an adversarial attack. You cannot wait for an incident to figure out how to unplug them. You need pre-approved "Degraded State" protocols.

1
Level 1: Surgical Feature Flagging

Scenario: A specific Agentforce Flow (e.g., "Draft Client Apology") starts behaving erratically.
Action: Disable the specific intent mapping via LaunchDarkly (or internal feature flags) without bringing down the rest of the CRM chat functionality. The UI gracefully degrades: "This specific action is currently under maintenance."

2
Level 2: Model Hot-Swapping

Scenario: A new vendor model update breaks the underlying reasoning logic enterprise-wide.
Action: Using your AI Gateway (from Tab 9), execute a single-click rollback. Route all API traffic back to the previous stable model version (e.g., swap from Model_v3 back to Model_v2.5). Zero downtime for end users.

3
Level 3: The Global Kill Switch

Scenario: Massive data exfiltration detected via Confused Deputy Attack in the Vertex environment.
Action: Sever all MCP (Model Context Protocol) server connections to internal databases. The LLMs can still chat, but they are instantly isolated from enterprise data. Trigger pre-written mass-comm to all 40,000 employees: "Our AI assistants are currently undergoing emergency security maintenance. Please revert to manual processing."

Human Dynamics

Front-Line Change Management

The technology is only 20% of the battle. Your 40,000 employees represent the true bottleneck. You must proactively manage the two psychological extremes of AI adoption.

Algorithm Aversion

The Profile: Typically tenured veterans. They refuse to use the agent because "a machine doesn't know my clients."

The Fix: Implement Confidence Scores in the UI. Instead of the AI demanding an action, it suggests: "Based on 3 recent emails (linked here), I am 92% confident this client needs an address update." Show the math to earn trust.

Automation Complacency

The Profile: Typically junior employees. They blindly trust the AI and aggressively hit "Approve" on every HITL prompt without reading it.

The Fix: Engineer Productive Friction. If an Agentforce Flow alters critical financial data, force the user to manually re-type a key value (like the client's last name) before the "Execute" button unlocks.

The Evals Feedback Loop

Every Agentforce and Vertex interface must have an aggressive, simple feedback mechanism (Thumbs Up / Thumbs Down / Edit). If a user edits an AI-drafted email before sending it, that diff (the difference between the AI draft and the human edit) must be automatically piped back into your LangSmith Evals dashboard. The human workforce becomes your distributed QA team, continuously refining the system prompts.
